DNS или Domain Name System «система доменных имён» - система позволяющая соотнести полное доменное имя (например google.com) с ip адресом (идентификатором)
для получения доступа к сетевым ресурсам (веб-сайт, хранилище данных и т.п.)

HTTP или HyperText Transfer Protocol «протокол передачи гипертекста» - протокол который посредством запросов клиента позволяет получать данные от сервера

HTTPS или HyperText Transfer Protocol Secure« защищенный протокол передачи гипертекста» — расширение протокола HTTP внедряющая шифрование передаваемых данных 
при помощи криптографических протоколов

HTTP/0.9 – первая версия протокола, определила взаимодействие между клиентом и сервером в сети.

HTTP/1.0 –  стал основой для реализации приложений и компонентов с использованием HTTP

HTTP/1.1 – исправление ошибок, доработка документации. внедрен режим постоянного соединения (можно отправлять и принимать запросы и ответы в рамках одного соединения).
Также теперь при установке соединения клиент должен отправлять в специальном поле имя хоста

HTTP/2.0 – добавлена возможность серверу отсылать дополнительные файлы которые еще не были запрошены пользователем, которые потребуются браузеру для отображения страницы.
Также было добавлено мультиплексирование (объединение) запросов с определением приоритетов и загрузкой элементов параллельно. 
Добавлена поддержка push-уведомлений от сервера.
